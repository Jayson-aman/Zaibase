# Zaibase 法務1枚サマリー（§19＋§20＋§21）

> **用途**：建設業・労働法専門の弁護士／社労士への初回相談（1〜2時間）用。  
> **免責**：本書は設計メモであり法律助言ではない。確認完了までは手数料本番徴収・組合名義正式契約を行わない。  
> **詳細正本**：[`teams-coop-design.md`](../teams-coop-design.md) ／ [`subcontract-chain-design.md`](../subcontract-chain-design.md) ／ [`SPEC.md`](../SPEC.md) §19・§20

---

## 1. Zaibaseとは（30秒）

建設業の一人親方マッチングプラットフォーム。**契約当事者ではない**。マッチング・証跡・決済の**透明性の担保者**。

---

## 2. 二軌制（§15④＋§19）— 混同禁止

| | **軌道1：個人** | **軌道2：組合** |
|---|---|---|
| **契約当事者** | 一人親方 ↔ 発注者 | **労働者協同組合** ↔ 発注者 |
| **法的根拠** | スキームB（業務委託仲介） | 労働者協同組合法（2022/10/1〜） |
| **典型規模** | ¥30万〜¥100万 | ¥500万〜¥2,000万 |
| **データ** | `bidType: individual` | `bidType: team_only` |
| **手数料** | 発注者負担・逓減（§6.4） | 同左（組合員賃金から天引きしない） |
| **内部分配** | — | **従事分量配当**（CCUS・稼働記録が根拠） |

```
【軌道1】発注者 ←──請負・業務委託──→ 一人親方
              └── Zaibase（プラットフォームのみ）

【軌道2】発注者 ←──請負契約──→ 労働者協同組合（登記済み）
              └── Zaibase（証跡・エスクロー・分配計算）
                        └── 組合員へ従事分量配当
```

**前提（軌道2）**：Zaibase上の「チーム」≠ 法人格。`legalStatus: registered_coop`（登記済み）のみ正式応募・組合名義契約可。

## 3. 多層チェーン（§20）— 大手→中小→組合→一人親方

```
大手ゼネコン（元請）
　↓ 元請契約
中小建設会社（1次下請）← 工程・品質・安全管理を実質担当
　↓ 専門工事を分離発注（丸投げ禁止）
労働者協同組合（2次下請）← タイル・配管等を実際に施工
　↓ 組合内業務分担
組合員の一人親方 ← 実際に手を動かす
```

| 論点 | 整理 |
|---|---|
| **一括下請負**（建設業法第22条） | 全部・主要部分の丸投げ禁止。違反で**営業停止15日以上** |
| **階層の深さ** | 明文上限なし。**実質3次が限界**。Zaibaseは **4次目をブロック** |
| **実質的関与** | 中間層は施工計画・工程・品質・安全・技術指導を担う。**名義貸しは違法リスク** |

**合法** ＝ 各層が実質関与 ＋ 専門工種の分離発注  
**違法** ＝ 中間層が丸投げ・名義だけ・中間搾取

## 4. Zaibaseの役割と禁止事項

### 4.1 やること（透明性の担保）

| # | 機能 |
|---|---|
| ① | 契約の透明化（階層・当事者・工種） |
| ② | 実質関与の記録（現場報告・安全報告・タイムスタンプ） |
| ③ | 下請構造の可視化（3次アラート） |
| ④ | 報酬の透明化（階層別・末端分配率・60日管理） |
| ⑤ | CCUS就業履歴の連携 |

### 4.2 絶対にしないこと

| ✗ 禁止 | 理由 |
|---|---|
| Zaibaseが中間に入り丸投げしながら手数料を取る | 商業ブローカー化・一括下請助長 |
| 4次以下の多重下請を容認・助長 | 監督不能・末端の単価剥奪 |
| 「名義だけ貸す」業者の登録 | 実質関与なし |
| 末端への不適正報酬を放置 | 下請法・フリーランス保護法 |
| 組合員報酬から職業紹介手数料を徴収 | 職安法・建設業は有料職業紹介対象外 |
| 出資口数に比例した配当計算 | 労働者協同組合は非営利法人 |
| 組合の設立・登記・労務の代行 | 有資格者法（弁護士・社労士・司法書士領域） |

### 4.3 手数料の取り方（中間業者にならない）

- **各マッチングリンクの発注側**から1回だけ（階層ごとに中抜きしない）
- ラベル：**プラットフォーム利用料**（職業紹介手数料ではない）
- 逓減率：〜100万8%／〜500万5%／〜2,000万4%／2,000万〜2.5% ＋ エスクロー管理1〜2%（§6.7）
- β期間：手数料徴収**凍結**（§6.2）

## 5. 確認質問一覧（弁護士・社労士へ）

### A. 二軌制・手数料（§15・§19）

| # | 質問 |
|---|---|
| **A1** | 軌道1（スキームB）で有料職業紹介許可なく運営できるか |
| **A2** | 軌道2のプラットフォーム手数料（発注者負担・逓減率）の**職業安定法**上の位置づけ（建設業務は有料職業紹介対象外） |
| **A3** | 軌道1と軌道2の**UI・契約書・手数料の分離**方針の十分性 |
| **A4** | 有料職業紹介許可申請が必要と判断される場合の整理（申請は建設業務には使えない） |

### B. 労働者協同組合・チーム（§19）

| # | 質問 |
|---|---|
| **B1** | `informal`／`forming`／`registered_coop` のシステムゲートで偽装組合・職業紹介リスクを抑えられるか |
| **B2** | エスクロー → 組合Connect口座 → 組合員分割Transferの**資金決済法**・組合内規上の要件 |
| **B3** | **従事分量配当**をCCUS就業日数で算定する妥当性（**社労士**） |
| **B4** | チーム信用スコア（0〜100）表示の**景品表示法・優良誤認**リスク |
| **B5** | 設立チェックリスト提供が**有資格者法**に触れない文言範囲 |

### C. 多層下請・透明性（§20）

| # | 質問 |
|---|---|
| **C1** | 専門工事「分離発注」の `tradeScope` 設計で一括下請（第22条）リスクを抑えられるか |
| **C2** | 実質関与5項目（施工計画・工程・品質・安全・技術指導）と証跡の妥当性 |
| **C3** | **3次で契約作成をブロック**する強制の可否・利用規約上の十分性 |
| **C4** | 階層別報酬表示が**下請法・フリーランス保護法**の書面要件を補完できるか |
| **C5** | Zaibaseが「透明性のみ」提供し**中間業者にならない**利用規約表現 |
| **C6** | 末端分配率アラート閾値の根拠 |
| **C7** | 多層チェーン全体を見る `#chainScreen` 実装の優先度と法的要否 |

### D. 登録ゲート・コンプライアンス（§21）

| # | 質問 |
|---|---|
| **D1** | 社保未加入者の「小規模のみ参加可」猶予設計の可否 |
| **D2** | 信用スコア（100点）表示の**景品表示法**・優良誤認 |
| **D3** | 反社チェックを自己申告のみで足りるか、第三者API要否 |
| **D4** | 破産・復権の自己申告とゲート拒否の妥当性 |
| **D5** | 書類期限切れによる応募停止・アカウント停止の利用規約根拠 |
| **D6** | 公共工事の社保・CCUS必須化の根拠十分性 |

## 6. ゲート通過の目安（相談後）

以下が整理できたら `platformConfig/teams` を更新：

```
lawyerGatePassed: true
lawyerConfirmedAt: "YYYY-MM-DD"
lawyerFirm: "（事務所名）"
schemeBConfirmed: true   // 軌道1
coopTrack2Confirmed: true // 軌道2
chainTransparencyConfirmed: true // §20
```

**ゲート通過後に解禁**：チーム専用案件の公開・登記済み組合の応募・本番手数料・組合名義契約・エスクロー本番。

**ゲート通過前でも可**：チーム結成（`createTeam`）・招待参加・分配シミュレーター。
