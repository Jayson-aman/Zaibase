## 0. これは何か（Why）

建設業界の **資材不足・人手不足・余剰資材の廃棄** を解決するプラットフォーム。
ナフサ・石油不足による資材難への対応も目的。職人（個人事業主）同士の **互助** を軸にする。

- アプリ名: **Zaibase（ザイベース）**
- キャッチコピー: 「Connect. Build. Share.」

## 4. 新機能（本セッションで追加・要件定義）

### 4-1. 生活ライン保証マッチング（労力シェア）
職人が「30日のうち最低何日働けば生活できるか（生活ライン）」を登録。
- 生活ラインに**未達の人へ優先的に仕事を回す**
- 仕事が余る人は**余剰を不足する人へシェア**
- 全員が生活ラインを下回らないようにする互助設計
- 優先対象: 出戻り（仕事が飛んだ人）・無職状態の人・一人親方

**設計メモ（要決定）**
- 仕事の出どころ: 職人同士の応援 / 元請の仕事 / 両方 →（ヒアリング中）
- 稼働日数のカウント: 自己申告 / アプリ自動 / 両方 →（ヒアリング中）
- 生活ラインの公開範囲: 非公開 / 状態のみ / 公開 →（ヒアリング中）

### 4-2. スポット応援マッチング（空き時間 × 移動時間）
「15:30から空き」のように空き時間を登録 → 近くの応援募集現場へ。
- **3時間前に告知メール**を自動送信
- **時給換算**で参加
- **Google Maps で移動所要時間を算出** → 実働可能時間と想定報酬を自動計算
  - 例: 15:30空き／現場18:00まで／移動35分 → 16:05着 → 約1h55m稼働 → 時給×時間 で報酬目安
- **遠い現場は表示しない**（距離フィルター）

**設計メモ（要決定）**
- 通える範囲: 30km / 50km / 80km / 車で◯分 / 職人ごとに設定 →（ヒアリング中）
- ⚠️ コンプラ: 「時給で時間管理」は雇用・派遣に寄るリスク。**時間単価の業務委託（指揮命令なし）** として設計し専門家確認。

### 4-3. 役所手続きサポート（システム化）
職人の **社会保険・税務署届出** を Zaibase で支援。
- 対象: 開業届 / インボイス登録番号 / 労災（特別加入）/ 社会保険 / 確定申告
- ⚠️ 現実: 「役所へ全自動送信」は不可。税理士法・社労士法で代行は有資格者のみ。
- 取れる道（3段階）:
  1. **書類作成＋手続きガイド＋期限リマインド**（すぐ可能）
  2. e-Tax / マイナポータル / 会計ソフト（freee・マネーフォワード）API連携
  3. 提携税理士・社労士への依頼窓口（合法的に代行）
- ①で作った書類が「登録ゲート」の証明にもなる。

### 4-4. 多言語対応
対応言語: 🇯🇵日本語 / 🇻🇳ベトナム語 / 🇰🇭カンボジア語（クメール語）/ 🇮🇩インドネシア語 / 🇬🇧英語
- **画面UIの文字**: 言語ごとの翻訳ファイル（i18n）で即時切替
- **ユーザー投稿の自由文（出品・チャット）**: **Vertex AI（翻訳/Gemini）で自動翻訳**
- 画面上部に 🌐 言語切替

## 6. 手数料体系（改訂案 v3 — 二層構造・2026年6月）

> **労務（スポット応援）** と **請負工事（専門工事）** で手数料を分離する。請負は単価が桁違いに大きいため**逓減方式**。旧案（一律16〜25%）・v2一律8%は廃止。  
> アプリ内シミュレータ：マイページ「💰 手数料シミュレータ」

### 6.1 設計思想（なぜ二層＋逓減か）

| 比較 | 労務マッチング | 請負工事 |
|---|---|---|
| 典型単価 | ¥8万/件 | ¥300万/件 |
| 手数料率 | 10%（職安法上限10.8%内） | **4%前後（逓減）** |
| 1件収益 | ¥8,000 | **¥120,000**（15倍） |

**逓減にする理由**

1. **発注者離脱防止** — 5,000万×8%＝400万は現実的でない  
2. **大型案件の誘引** — 「大きいほどお得」で元請・大手の登録を促す  
3. **不当利益リスク回避** — 工事代金に対する高率徴収の問題を避ける  

**役割分担**

- **労務** ＝ ユーザーを集める入口（件数・認知）。単価は低いが職安法枠内  
- **請負** ＝ 収益の柱。件数は少なくてよい  

### 6.2 フェーズ別課金

| フェーズ | 手数料 |
|---|---|
| **β（現在）** | **0%**（許可・弁護士確認まで凍結） |
| **許可取得後** | 下表（労務・請負それぞれ届出・契約上の整理が必要） |

### 6.3 労務マッチング手数料（スポット・小額請負）

有料職業紹介に該当する場合の枠。**上限10.8%**。

| 項目 | 値 |
|---|---|
| 標準率 | **10%** |
| プロ会員 | **9%**（−1%） |
| マックス会員 | **8%**（−2%） |
| 最低手数料 | **¥500/件** |
| 法定上限 | 報酬 × **10.8%** |

### 6.4 請負工事手数料（逓減・専門工事直接請負）

**法的整理（要弁護士確認）**：「有料職業紹介」ではなく**工事請負契約の仲介手数料**として設計。現行モデルC（専門工事分離・直接請負）が前提。

| 請負金額（税抜想定） | 手数料率 |
|---|---|
| 〜¥100万 | 8% |
| ¥100万〜¥500万 | 5% |
| ¥500万〜¥2,000万 | **4%** |
| ¥2,000万〜 | **2.5%** |

| 会員割引 | 全ティアから −0.5%（Pro）/ −1.0%（Max） |
|---|---|
| 最低手数料 | ¥3,000/件 |

**例**：¥300万請負 → 4% ＝ **¥120,000/件**

### 6.5 月次収益シナリオ（デフォルト試算）

| 収入源 | 前提 | 月額 |
|---|---|---|
| 労務手数料 | 50件 × ¥8万 × 10% | 約40万円 |
| 請負手数料 | 10件 × ¥300万 × 4% | **約120万円** |
| サブスク | Pro50×¥3,980＋Max20×¥6,980 | 約34万円 |
| Stripe等（控除） | 決済導入時 | 約▲21万円 |
| **売上合計** | | **約173万円** |
| 固定費 | インフラ8万＋人件50万 | 約58万円 |
| **営業利益** | | **約115万円** |

※ **請負10件の有無で収益が天地ほど変わる**。労務は入口、請負が本丸。

### 6.6 資材フリマ（別枠・第2期）

| 区分 | 手数料 |
|---|---|
| 売買・レンタル | 5% / 8%（金額帯で逓減可） |
| 物々交換 | ¥300/件 |

### 6.7 エスクロー（第4期・任意）

請負手数料に **+1〜2%** の管理料を上乗せ検討。合算でも発注者が離脱しない逓減設計を維持。

### 6.8 廃止した旧案

| 案 | 問題 |
|---|---|
| 16〜25%一律 | 職安法・炎上・離脱 |
| v2 一律8% | 請負の収益ポテンシャルを過小評価、労務と混同 |

## 7. サブスクリプション（改訂案 v4）

**2軸モデル**：① **受注者（職人）向け** Pro/Max（本章） ② **発注者（元請・現場）向け**（§7.5・別料金でARPU向上）

### 7.0 解約されにくい設計の核心

最強の解約防止は **「やめると損になる構造」**。

| 軸 | 仕組み | Zaibaseでの実現 |
|---|---|---|
| **手数料割引の自走** | 払うほど儲かる | Maxの請負−1%は単価が大きいほど効く。300万×3件/月＝**月9万円節約**、月額6,980円差し引きで**約+8.3万円**の合理的継続 |
| **データ蓄積ロック** | 記録が資産になる | 案件履歴・取引先・評価（第2期：GPS出退勤・CCUS実績）が積み上がるほど、解約＝「自分の仕事の記録が見えなくなる」心理的コスト＋競合移行コスト |

サブスク単体（試算約34万/月）は請負手数料（約120万/月）の補助輪。**本質価値は手数料ROI＋マッチング成功率＋蓄積データ**。

### 7.2 フリープラン（ベース）の設計思想

**体験版ではなく「不便な実用版」**。「無料で十分」ではなく **「無料だと損」** を体感させる。

| 制約 | 内容 | 意図 |
|---|---|---|
| **応募制限** | 月**5件**まで（案件への応募・問い合わせ） | 使えるが本格利用には足りない |
| **手数料+2%** | 労務・請負とも標準率に+2%上乗せ | 取引が増えるほど有料化のメリットが明確 |
| 労務の上限 | +2%後も **職安法10.8%上限**でキャップ | 法適合。請負側の+2%が主な損失体感 |

例：請負300万の場合、ベース6%（4%+2%）＝18万 vs Max 3%（4%−1%）＝9万 → **1件で9万円差**。

投稿（案件登録）はベースでも可。制限は **応募側** に置く（受注者のフリープラン設計）。

### 7.4 解約防止設計

#### A. 解約時に失うもの

| 資産 | ダウングレード時 |
|---|---|
| 手数料優遇 | +2%（ベース）に戻る → 請負1件で数万〜十万の差 |
| バッジ・優先枠 | 信頼シグナル・問い合わせ減 |
| 応募無制限 | 月5件に制限 |
| 実績・履歴の可視化 | 3件表示・1年制限 |
| 蓄積データ（第2期） | GPS/CCUSは閲覧・エクスポート不可 |

**解約確認UI**：「過去12ヶ月の節約額 ¥○○」「非表示になる実績 ○件」「ベースに戻すと手数料+2%（今月換算 ¥○○増）」を必ず表示。

#### B. 継続インセンティブ

| 施策 | 内容 | 期 |
|---|---|---|
| 節約額レポート | マイページ常時「今月¥○○節約中」 | 第1期 |
| しきい値通知 | 請負2件目で「Maxの方が¥○○お得」 | 第2期 |
| 年払い | 10ヶ月分で12ヶ月（2ヶ月無料） | 決済時 |
| 一時停止 | 年1回×1ヶ月、バッジ・実績維持 | 第2期 |
| データ保持 | 退会後90日は再入会で復元 | 第1期 |

#### C. 解約フロー

```
解約 → 節約額・+2%損失・失う実績を数値表示
     → 理由選択（料金/未使用/他社/機能不足）
     → 代替案（一時停止 / Pro↔Max / 通知設定見直し）
     → 最終確認
```

#### D. 4段階通知タイムラインと狙い

UI正本：`frontend/Zaibase.html` ／ バックエンド：`functions/churn_prevention.js`

| 段階 | タイミング | チャネル | 狙い |

| **1** | 解約ボタン押下直後 | アプリ内モーダル | **最重要**。「¥◯万受注・¥◯万節約」の金額で「やめてよかったのか？」の迷いを生む |

## 9. コンプライアンス（最重要）

> ⚠️ 本項は一般知識。**ローンチ前に必ず社労士・弁護士のレビュー必須**。

### 労働関連（労力シェア・応援の肝）
- **建設業の労働者派遣は原則禁止** → Zaibaseは**派遣にしない**
- すべて **業務委託（請負）** のマッチングとする（雇用でも派遣でもない）
- **偽装請負の防止**: 発注者は直接の **指揮命令をしない** ことを契約・誓約書に明記
- **職業安定法**: 人材紹介に該当する形態は許可が必要（有料職業紹介事業許可）→ 形態を専門家確認
- **労基法・人権**: 中間搾取・多重委託・長時間 を制限。報酬の透明化。
- **時給制の注意**: 時間管理は雇用・派遣に寄るため「時間単価の業務委託」として設計

### 誓約書システム（実装予定）
1. 登録ゲート: 一人親方/個人事業主であること、**開業届・インボイス番号・労災（特別加入）** を必須
2. 電子誓約書: 仕事受注前に両者が同意（派遣でない/指揮命令しない/法令順守）
3. 同意ログ保存: 誰が・いつ・どの版に同意したかを Firestore に記録
4. 未同意なら応募ブロック

### 外国籍ユーザーの在留資格確認（不法就労助長の防止）
- リスク: 就労資格の確認を怠ると **不法就労助長罪**。**仲介したZaibaseも処罰対象**になり得る → 確認は必須
- 確認項目: 在留カードの真贋（入管庁の失効情報照会・ICチップ読取）、在留資格、就労可否・範囲、在留期限、資格外活動許可（留学等は週28時間）
- ⚠️ **技能実習・特定技能は原則 自由な副業/スポット就労/転職ができない** → 労力シェア・スポット応援は使えない可能性大
- 対策: 在留資格でフィルター。**自由に就労できる資格（永住者・定住者・日本人/永住者の配偶者等）のみ**を労力シェア対象に
- システム: 外国籍は在留カードのアップロード必須 → 資格判定で対象業務を制限 → 在留期限を記録しリマインド・期限切れ自動停止 → 記録保存（9-9と共通）
- 要・入管専門の行政書士/弁護士確認

### 物品・取引
- 汚染土の出品を完全ブロック（土壌汚染対策法）
- 鋼材はミルシート必須
- 廃材・使用不能資材の偽装防止（写真複数枚・状態選択必須）

### 役所手続き
- 税理士法・社労士法: 代行は有資格者のみ（提携で対応）
