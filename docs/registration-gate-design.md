## 0. なぜ登録ゲートが最大リスクか

Zaibaseはマッチングの**入口**を握る。未加入・無許可・反社・多重下請の当事者を通すと、**発注者（元請）が法令違反に巻き込まれる**。プラットフォームは「透明性の担保者」であると同時に、**参加資格のフィルター**を担う設計が必要。

## 1. 抜け穴の全体マップ（6カテゴリ）

| # | カテゴリ | 法的根拠・ガイドライン | Zaibaseへの影響 |
|---|---|---|---|
| **①** | **社会保険・保険未加入** | 国交省「社会保険の加入に関する下請指導ガイドライン」— 未加入は下請に選定すべきでない・入場認めない。2020年〜許可新規・更新にも影響 | 未加入者を通すと**発注者が違反リスク** |
| **②** | **破産・欠格要件** | 建設業法の欠格事由（破産者で復権なし等）。復権済み（免責後3〜6ヶ月）は申請可 | 確認なし登録→発注者が**不知で契約** |
| **③** | **暴力団・反社** | 欠格事由：役員等が暴力団員・関係者。過去5年以内の該当も | 許可取得・更新不可。プラットフォームから**排除必須** |
| **④** | **下請構造** | 明文の次数上限なし。元請の指導監督義務。自治体は**3次制限**多い | §20 `tierLevel > 3` ブロック |
| **⑤** | **一括下請負（丸投げ）** | 建設業法第22条 — 全部・主要部分の丸投げ禁止。違反で**営業停止15日以上** | 専門工種分離・実質関与証跡 |
| **⑥** | **偽装一人親方** | 実質雇用なのに一人親方扱い。社保・労働法逃避 | スキームB設計・指揮命令排除・組合は実態ある運用 |

### カテゴリ間の関係

```
登録ゲート（①②③⑥）  →  「誰を通すか」
案件参加ゲート（①③＋規模）  →  「どの案件に出せるか」
多層チェーン（④⑤）  →  「契約構造が合法か」（§20）
信用スコア  →  ①②③⑥の証跡を数値化
```

## 2. 登録ゲート：入れる人・入れない人

### 2.1 個人（一人親方）

| 判定 | 条件 |
|---|---|
| ✅ **通過（基本）** | 本人確認（KYC）完了 |
| ✅ **推奨** | 一人親方労災（特別加入）証明、CCUS技能者ID、開業届 |
| ✅ **公共工事・大規模** | 社会保険加入証明（国保・建設国保・国民年金等）**必須** |
| ❌ **拒否** | 社会保険未加入（公共・大規模案件向け）、破産中（復権前）、反社該当、許可取消から5年未満 |
| ⚠️ **猶予** | 社保証明未提出→小規模案件のみ・スコア上限。期限付きで提出促進 |

### 2.2 法人（中小建設会社・組合）

| 判定 | 条件 |
|---|---|
| ✅ **通過** | 建設業許可証、社会保険加入証明（健保・厚年・雇保）、法人登記簿、納税証明 |
| ❌ **拒否** | 許可なしで500万超受注意図、反社、欠格事由該当 |

### 2.3 破産・復権の整理

| 状態 | 登録 |
|---|---|
| 破産手続中・復権前 | ❌ |
| 免責・復権済み | ✅（自己申告＋必要なら書類。Zaibaseは法務機関ではない） |

## 3. KYC：登録時に求める証明（設計）

### 3.1 個人（一人親方）

| 書類 | 必須度 | 検証 |
|---|---|---|
| 身分証明書 | **必須** | Phone Auth＋画像アップロード（将来OCR） |
| 開業届写し | 推奨 | 手動確認→§4-3と連携 |
| 一人親方労災加入証明 | 推奨（公共工事で必須） | 有効期限管理 |
| CCUS技能者ID | 推奨（公共工事で必須） | `registerCcusWorkerId` 連携済み |
| 社会保険加入証明 | 条件付き必須 | 有効期限管理 |

### 3.2 法人

| 書類 | 必須度 |
|---|---|
| 建設業許可証 | **必須** |
| 社会保険加入証明 | **必須** |
| 法人登記簿謄本 | **必須** |
| 納税証明書 | 推奨 |

**原則**：Zaibaseは**書類保管・期限管理・ゲート判定**。許可の真偽最終確認は発注者・行政。反社チェックは**提携サービスまたは自己申告＋利用規約**（弁護士確認要）。

## 4. Zaibase信用スコア（100点）

| 項目 | 配点 | データソース |
|---|---|---|
| 社会保険加入 | 20 | `verificationDocs.socialInsurance` |
| 建設業許可 | 20 | `profileLicenseNo`（一人親方は小規模のみ**免除可**） |
| CCUS登録 | 15 | `craftsmanProfiles.ccusWorkerId` |
| 一人親方労災加入 | 15 | `verificationDocs.workersComp` |
| 取引実績・評価 | 20 | `contracts` 完了・レビュー |
| 本人確認完了 | 10 | Phone Auth + KYCステータス |
| **合計** | **100** | `complianceScore`（非正規化） |

**表示注意**：景品表示法・優良誤認（§19.7 B4）。「参考指標」表記・発注者の最終判断は当事者。

## 5. 案件参加ゲート（規模別）

| 案件条件 | 参加要件 |
|---|---|
| **¥500万未満** | 建設業許可なしの一人親方も可（スキームB・業務委託） |
| **¥500万以上** | **建設業許可必須** |
| **公共工事を含む** | 社会保険加入証明**必須**、CCUS登録**必須** |
| **¥2,000万以上（大型）** | **信用スコア70点以上**、チーム案件は `registered_coop`（§19） |
| **bidType: team_only** | 法務ゲート + 登記済み組合（§19） |

**実装**：`helpPosts` 作成時に `requiredCompliance` フィールド、`applications` / `teamApplications` 作成時に Functions で検証。

## 6. 下請層数・一括下請（§20連携）

| tierLevel | 動作 |
|---|---|
| 1〜3次 | 通常OK（実質関与・専門分離を満たすこと） |
| **4次** | ⚠️ アラート＋発注者書面承諾必須（推奨） |
| **>3次（システム既定）** | **契約作成ブロック** |

一括下請：上位と下位の工事範囲が同一・中間層の `substantiveInvolvement` ゼロ → ブロック／管理者通知。

## 7. 証明書有効期限管理

| タイミング | 動作 |
|---|---|
| 期限30日前 | Push／メールリマインド |
| 期限切れ | **案件応募自動停止** |
| 期限切れ90日 | **アカウント一時停止**（再提出で解除） |

対象：建設業許可、社会保険、労災特別加入、インボイス（任意）。

## 8. Firestore スキーマ（概要）

### `craftsmanProfiles` 拡張

```typescript
{
  businessType: 'hitori' | 'kojin' | 'houjin';
  complianceScore: number;           // 0-100
  complianceScoreUpdatedAt: Timestamp;
  gateStatus: 'pending' | 'limited' | 'verified' | 'suspended';
  kycStatus: 'none' | 'phone' | 'docs_submitted' | 'verified';

  verificationDocs: {
    socialInsurance?: { status, expiresAt, storagePath, verifiedAt };
    workersComp?: { status, expiresAt, storagePath, verifiedAt };
    constructionLicense?: { number, expiresAt, verifiedAt };
    bankruptcyDeclaration?: { selfDeclared: boolean, rehabilitated: boolean, note };
    antisocialDeclaration?: { selfDeclaredClean: boolean, declaredAt };
  };

  participationFlags: {
    canApplyUnder5M: boolean;
    canApplyOver5M: boolean;
    canApplyPublicWorks: boolean;
    canApplyLarge: boolean;          // 20M+, score>=70
  };
}
```

### `complianceAlerts/{id}`

期限切れ・ゲート拒否・tier警告のログ（運営・ユーザー通知用）。

### Cloud Functions（予定）

| 関数 | 役割 |
|---|---|
| `recalcComplianceScore` | 書類・CCUS・実績からスコア再計算 |
| `checkApplicationGate` | 応募前の規模・スコア・書類チェック |
| `complianceExpiryScheduler` | 期限30日前／切れ／90日停止 |

## 10. 弁護士・社労士への確認（登録ゲート追加分）

| # | 質問 |
|---|---|
| **D1** | 社保未加入者の「小規模のみ参加可」猶予設計の可否 |
| **D2** | 信用スコア表示の法的表現（景表法） |
| **D3** | 反社チェックを自己申告のみで足りるか、第三者API要否 |
| **D4** | 破産・復権の自己申告とゲート拒否の妥当性 |
| **D5** | 期限切れによる応募停止・アカウント停止の利用規約上の根拠 |
| **D6** | 公共工事案件の社保・CCUS必須化の根拠十分性 |

※ D1〜D6 は [`zaibase-legal-onepager-sec19-20.md`](./legal/zaibase-legal-onepager-sec19-20.md) の A〜C に追加して相談。

## 11. Zaibaseがやらないこと（再掲）

- 許可・社保の**行政審査の代行**
- 反社の**断定**（フラグ・申告・提携調査まで）
- 名義貸し・丸投げ構造の**黙認**
- スコアだけで「安全保証」と**表示しない**
