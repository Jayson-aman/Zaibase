## 1. 事業の概要

### 何をするサービスか

**Zaibase法律相談**は、建設職人・一人親方・下請け業者を主なユーザーとした、建設業・労働法特化の法律相談プラットフォームです。

ユーザーがスマートフォンから相談を送信し、弁護士・専門家が管理者パネルから回答する非同期型の法律相談サービスです。

### ターゲット

| ユーザー層 | 抱えやすい問題 |
|---|---|
| 一人親方・職人 | 未払い・遅延支払い、労災、長時間労働、偽装請負 |
| 下請け業者 | 下請代金の不当減額、一括下請け、書面交付の不備 |
| 元請・工務店 | 建設業法の許認可、契約書トラブル、フリーランス保護法対応 |
| 外国人技能実習生 | ハラスメント、在留資格、労働条件 |

## 2. サービス構成

### 相談の仕組み

```
① ユーザーがスマホから相談を送信（カテゴリ選択 + 詳細記述・最大3,000文字）
        ↓
② Firebase に保存 → 管理者パネルに通知
        ↓
③ 弁護士が管理者パネルにログインして相談を確認・回答入力
        ↓
④ ユーザーに「回答済み」通知 → アプリで回答を確認
```

### 相談カテゴリ

1. 未払い・遅延支払い（フリーランス保護法）
2. 下請代金・一括下請負（下請法）
3. 長時間労働・過重労働
4. 労災・業務上疾病
5. ハラスメント
6. 建設業法・許認可
7. 契約書・書面交付
8. その他

### 管理者パネルの機能（弁護士側）

- 未回答相談の一覧表示（カテゴリ・タイトル・相談内容）
- 回答入力（本文 + 「次のステップ」アドバイス欄）
- ステータス管理（回答待ち / 回答済み / クローズ）
- 招待コード：`ZB-LEGAL-BF97-0773-FFB1`

## 4. Zaibase Groupとの協力体制（提案）

川原総合法律事務所との協力関係として、以下のモデルを想定しています。

### 協力モデルA：回答専門家として参画

- 弁護士が管理者パネルにログインし相談に回答
- 回答はZaibase法律相談の名義で配信（弁護士名の表示は相談内容に応じて）
- 収益配分：月次の相談回答数・有料会員数に応じてZaibase Groupから事務所へ支払い

### 協力モデルB：顧問契約＋リード獲得

- 月定額の顧問料でZaibase法律相談の専門家として名義掲載
- プロプランユーザーへの「弁護士紹介機能」で事務所への案件送客
- リード（案件紹介）の成果報酬型報酬

### 協力モデルC：共同ブランディング

- 「Zaibase法律相談 powered by 川原総合法律事務所」としてサービスを展開
- 建設業専門の法律相談としての認知度向上

> 具体的な条件・契約形態はご要望に応じて協議します。

## 5. メリット（川原総合法律事務所にとって）

### ① 建設業特化の新規クライアント獲得

- Zaibase建設のユーザー（建設職人・一人親方・元請業者）が直接法律相談のユーザーになる
- 「建設業に強い弁護士」としての専門性ブランディングに繋がる
- 建設業・労働法分野の相談が中心なので、事務所の既存専門性とシナジーが高い

### ② 非同期・テキスト相談で業務効率化

- 電話・来所対応不要。管理者パネルから時間を選んでテキスト回答
- 相談内容がすべてデジタルで記録されるため、事例管理・再利用が容易
- 補助スタッフがトリアージし、重要案件のみ弁護士が対応する体制も可能

### ③ 定額収益（サブスク型）

- 従来の「案件ベース」報酬ではなく、月次定額収益が見込める
- 有料プランユーザーが増えるほど安定した月次報酬になる

### ④ 将来的な本格案件へのアップセル

- 「テキスト相談→面談・委任」の自然なフローを設計可能
- プロプランの「弁護士紹介機能」で事務所への直接受任に繋がる

### ⑤ 建設業界への社会的貢献

- 建設職人は法律的知識にアクセスしにくい層。相談窓口を持つことで社会的意義が高い
- 外国人技能実習生・特定技能労働者向けの多言語対応も可能（将来）

## 6. リスク

### ① 弁護士法第72条（非弁行為・周旋禁止）

**内容**：弁護士でない者が報酬を得る目的で法律事務の周旋・仲介を行うことを禁止。

**具体的なリスク**：
- Zaibase Groupが「弁護士への相談を仲介して手数料を取る」構造が非弁行為にあたるリスク
- 「Zaibase建設 → 法律相談への誘導」が周旋・紹介とみなされるリスク

**現在の対応**：
- Zaibase建設と法律相談のサービス間で商業的な誘導文言は使用していない
- 建設会員向けグループ割引は法的確認が取れるまで停止中
- 「法律相談はこちら」のような誘導リンクは現在設置していない

**確認が必要なこと**：
- Zaibase Groupが月額料金を受領し、弁護士への費用を支払うスキームの適法性
- 「弁護士紹介機能」の設計要件（弁護士紹介料の受領は弁護士法違反か）

### ② 守秘義務・情報管理

**内容**：弁護士の守秘義務（弁護士法第23条）とデジタルプラットフォームでの情報管理の整合性。

**具体的なリスク**：
- 相談内容がFirebase（Google Cloud、東京リージョン）に保存されること
- 運営者（Zaibase Group）が技術的に相談内容にアクセスできる状態
- データ漏洩時の責任の所在

**現在の対応**：
- Firebaseのセキュリティルールで本人と管理者のみアクセス可能に制限
- プライバシーポリシーに相談内容の取扱いを明記
- 管理者アクセスは招待コード制（`ZB-LEGAL-BF97-0773-FFB1`）

### ③ 応答品質・責任範囲の不明確化

**内容**：テキスト回答の内容が「法的助言」として受け取られた場合のリスク。

**具体的なリスク**：
- ユーザーがテキスト回答を元に行動し損害を被った場合の責任
- 情報が古い・不正確な場合の対応

**現在の対応**：
- 回答画面に「本回答は一般的な情報提供であり、法律上の助言ではありません」の免責事項を表示
- 「次のステップ」欄で「○○機関にご相談ください」と誘導

### ④ 応答時間へのプレッシャー

**内容**：ユーザーが支払いをしているため、回答が遅れると不満・解約につながる。

**具体的なリスク**：
- プロプラン（¥7,800/月・無制限）ユーザーが集中した場合の対応負荷
- 緊急性の高い相談（労災・解雇通知等）への即時対応が難しい

## 7. デメリット

### ① 非同期テキストの限界

- 電話・面談と比べて情報収集に限界がある（最大3,000文字のテキストのみ）
- 証拠書類・契約書の確認ができない（添付機能なし・将来的に追加予定）
- 緊急案件（即日解雇・労災事故）にはリアルタイム対応できない

### ② 弁護士受任への転換率の不確実性

- テキスト相談で解決してしまうとアップセルに繋がらない可能性
- ユーザーが「月¥7,800払えば弁護士レベルの回答が得られる」と誤解するリスク

### ③ 収益配分の規模感

- 現在はβ段階でユーザー数が少ない。初期収益は限定的。
- ユーザー数が増えるまでの立ち上げ期間は費用対効果が低い可能性

### ④ プラットフォーム依存

- Zaibase Groupがサービスを停止・方針変更した場合のリスク
- 協力関係の契約書・SLA（サービスレベル）の整備が必要

## 8. 現時点の法令対応状況

| 法令 | 対応内容 |
|---|---|
| 弁護士法第72条 | 非弁行為・周旋禁止。グループ割引・弁護士紹介料は確認が取れるまで停止。 |
| 個人情報保護法 | プライバシーポリシー整備。相談内容はFirestoreに暗号化保存。 |

| 弁護士法第23条（守秘義務） | 技術的アクセス制限（本人・招待管理者のみ）。運営担当者のアクセスポリシー要整備。 |
