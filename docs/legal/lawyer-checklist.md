作成：2026年6月21日  
運営：南條 雅哉 / Zaibase Group  
連絡先：info@zaibase.group  
アプリURL：https://zaisai-share.web.app/app

## このドキュメントの目的

Zaibase建設は現在、機能の一部を「法的確認が取れていない」として意図的に停止しています。  
収益化が進んだ段階で順次弁護士に確認し、開放していく方針です。  
本書は、その確認を依頼するための論点整理です。

---

## サービス概要（弁護士への説明用）

- **何をするサービスか**：建設職人（一人親方・職人）と発注者（元請・工務店）を繋ぐプラットフォーム
- **現在の収益モデル**：月額プラン料金（フリー無料 / スターター¥1,980 / プロ¥5,480 / チーム¥14,800）
- **現在停止中の機能**：マッチング手数料課金・多層契約・グループ割引（詳細は以下）

---

## 確認事項①：有料職業紹介（最優先）

### 現状
- 職人と発注者が案件でマッチングする機能は実装済み
- 現在は「案件情報の掲示板」として無料で提供
- マッチング成立時に手数料を課金する機能は**停止中**（職業安定法の許可未取得のため）

### 確認したいこと
1. 現在の「案件掲示板」モデルは有料職業紹介に該当しますか？（無料の情報提供として適法か）
2. マッチング成立時に手数料（例：報酬の5〜10%）を受け取るには、有料職業紹介事業者の許可（職業安定法第30条）が必要ですか？
3. 許可取得の要件・費用・期間を教えてください
4. 「請負工事の仲介」と「労働者の職業紹介」はどこで区別されますか？（建設業者が現場ごとに職人を手配するケースが対象）

### 関連法令
- 職業安定法 第30条（有料職業紹介事業の許可）
- 職業安定法 第32条（手数料の制限）

## 確認事項②：多層契約・下請けチェーン管理機能

### 現状
- 「一次下請け → 二次下請け」の多段階の請負関係をアプリ上で管理・可視化する機能
- 弁護士確認が取れるまで**停止中**

### 確認したいこと
1. プラットフォームが多層の下請け関係を「管理・記録する」だけの機能は適法ですか？
2. 一括下請け（丸投げ）を禁止している建設業法第22条との関係で、当プラットフォームが仲介することでリスクはありますか？
3. 施工体制台帳（建設業法第24条の8）の作成補助はどこまで可能ですか？
4. 発注者・一次下請け・二次下請けの三者間で電子契約書を生成する機能を提供することの適否

### 関連法令
- 建設業法 第22条（一括下請けの禁止）
- 建設業法 第24条の8（施工体制台帳）
- 下請代金支払遅延等防止法（下請法）

## 確認事項③：弁護士法72条との関係（法令情報提供の範囲）

### 現状
- アプリ内に「許認可・法令チェック」「コンプライアンスゲート」「下請コンプライアンス」などの画面がある
- 内容は国土交通省ガイドライン・建設業法の条文紹介と自社チェックリスト
- 「弁護士に相談してください」と明示し、具体的な法的判断はしていない

### 確認したいこと
1. 現在の「法令情報の提供＋チェックリスト」は弁護士法72条（非弁行為）に抵触しませんか？
2. AIが「この案件は下請法に抵触する可能性があります」と表示することは問題ですか？
3. 将来的に弁護士を紹介する機能（弁護士紹介料）を設ける場合、何が必要ですか？

### 関連法令
- 弁護士法 第72条（非弁護士の法律事務の取扱等の禁止）

## 確認事項④：労災サポート（書類記入補助）

### 現状
- 労災申請書（厚生労働省様式）の記入支援機能がある
- 「入力補助であり、申請代理ではない」と明示
- 社労士への相談窓口へ誘導している

### 確認したいこと
1. 労災申請書の「記入補助」は社会保険労務士法第27条に抵触しますか？
2. 「書類を印刷して自分で提出してもらう」形式であれば問題ないですか？
3. 将来的に提携社労士が申請代理する機能を設ける場合の適切な構成は？

### 関連法令
- 社会保険労務士法 第27条（業務の制限）

---

## 確認事項⑤：グループ割引（建設プロプラン → 法律相談割引）

### 現状
- Zaibase建設のプロプラン会員が、別サービス「Zaibase法律相談」を¥1,000引きで利用できる割引機能
- 弁護士法72条（弁護士の周旋・紹介禁止）への抵触を懸念して**停止中**（管理者承認フラグ`groupDiscountGranted: false`）

### 確認したいこと
1. 建設アプリのプラン会員に法律相談サービスの割引を提供することは「弁護士の周旋・紹介」に該当しますか？
2. 「法律相談への誘導」とならない適切な表現・導線の設計を教えてください
3. グループ内の横断割引として適法に実施するには何が必要ですか？

### 関連法令
- 弁護士法 第72条
- 弁護士法 第73条（譲り受けた権利の実行を業とすることの禁止）

## 確認事項⑥：エスクロー決済・資金移動

### 現状
- 職人への報酬を、発注者がZaibase経由で先に預け、完了確認後に職人に支払うエスクロー機能が実装済み
- 現時点ではStripe Connect経由で実装（Stripe側でライセンス対応）

### 確認したいこと
1. Stripe Connect経由のエスクロー（資金の一時保管→送金）は、資金決済法上の「資金移動業」に該当しますか？
2. Stripe Connectを使う限り当社は登録不要ですか？それとも別途登録が必要ですか？
3. 将来的にZaibase独自の決済機能を持つ場合に必要な許可は？

### 関連法令
- 資金決済に関する法律（資金決済法）第37条（資金移動業の登録）

## 確認事項⑦：顔認証・GPS情報の取得

### 現状
- 現場入退場に顔認証（顔写真の登録・照合）とGPS位置情報を使用
- プライバシーポリシーに記載済み

### 確認したいこと
1. 顔認証データの取得・保存に特別な同意取得や安全管理措置は十分ですか？
2. 外国人技能実習生・特定技能の方のデータ取得に追加の配慮は必要ですか？
3. 個人情報保護委員会への届出が必要なケースはありますか？

### 関連法令
- 個人情報の保護に関する法律（個人情報保護法）
- 個人情報保護委員会「顔識別機能等を有するAIの利用に関するガイドライン」

## 優先順位の目安

| 優先度 | 確認事項 | 理由 |
|---|---|---|
| 🔴 最優先 | ①有料職業紹介 | 許可取得でマッチング手数料が取れるようになり、主要収益源になる |
| 🔴 最優先 | ⑤グループ割引 | 機能は完成済みで承認を待っているだけ |
| 🟡 次点 | ②多層契約 | 機能完成済み・開放待ち |
| 🟡 次点 | ③法令情報提供 | 現状でも動いているが、AIによる判断支援を強化したい |
| 🟢 余裕があれば | ④労災サポート | 現状は補助的な位置づけで大きなリスクは低い |
| 🟢 余裕があれば | ⑥エスクロー | Stripe Connectを使っている間はStripe側が対応 |
| 🟢 余裕があれば | ⑦顔認証・GPS | プライバシーポリシーは整備済み |

## 参考：現在の法令遵守状況

- 特定商取引法：記載済み（アプリ内「特商法表記」画面）
- 個人情報保護法：プライバシーポリシー整備済み
- 建設業法：施工体制台帳・見積書・契約書で準拠条文を明示
- 職業安定法：有料マッチング停止・免責事項明示済み
- 弁護士法：法的判断不提供の免責明示済み
- 社労士法：書類補助のみ・申請代理なしを明示済み
